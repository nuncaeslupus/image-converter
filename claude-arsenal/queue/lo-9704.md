## Acceptance gate
After a production build, the app loads and runs a full trace with the network
offline (SW serves the app shell + WASM from cache). Verified manually:
served `dist/` via a plain static server under the `/image-converter/` base,
loaded the page (SW registered, 7 precache entries incl. `.wasm` + worker),
killed the server, reloaded — the app rendered fully from cache.

```bash
npm run build && npm run test:build
```

## Implementation
- `vite.config.ts`: added `VitePWA({ registerType: "autoUpdate", workbox.globPatterns
  includes wasm, minimal manifest })`. WASM is not in workbox's default glob, so it
  must be added explicitly or the tracer won't work offline.
- `package.json`: `vite-plugin-pwa` devDependency.
- No app code change — `injectRegister: "auto"` injects `registerSW.js` at build.

## Notes
- `vite preview` mis-serves `sw.js`/`registerSW.js` as `text/html` (over-broad SPA
  fallback in vite 8) — offline can only be verified against a plain static server,
  which is also the real deploy target (GitHub Pages).
